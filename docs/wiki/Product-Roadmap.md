# Product Roadmap

## Now

- Keep CI green on every PR.
- Use the manual Release workflow for signed APK publishing.
- Keep the coverage dashboard visible in Actions artifacts.
- Preserve log redaction in all diagnostic flows.

## Next

- Decide the production iOS downloader strategy.
- Add a visible downloads history.
- Add clearer unsupported-site and rate-limit recovery messaging.
- Add branch protection requiring Mobile CI.
- Move these wiki-ready pages into the GitHub Wiki if GitHub Wiki is the chosen docs surface.

## Later

- Add update checks for yt-dlp engine/runtime versions.
- Add localization for key user-facing messages.
- Add privacy and app-store review documentation.
- Add a backend downloader mode if mobile runtime constraints become too risky.
